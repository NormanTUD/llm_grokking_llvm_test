2. Rubber Sheet — Watch the Grid Deform Layer by Layer

3. Guitar Strings — Token Trajectories Through Layers

Each "string" is one token, vibrating sideways as it passes through layers (from bottom to top). The X-axis shows position along one PCA dimension, the Y-axis shows layer depth.

When two prompts differ by one word, most strings stay overlapped — but at specific layers, specific strings peel apart. That peeling shows you exactly which layer and which geometric direction "cares about" the difference.
Divergence Plot — For each token position, this shows the distance between the two prompts' representations at each layer. Bright spots = where the model's internal representation diverges due to the changed word.

4. Heat Map of Caring — Which Tokens Get Worked On When

A simple but powerful view: brightness = how much the representation changed at that token at that layer (L2 norm of the residual delta).

Bright columns = tokens the model works hardest on. Bright rows = layers that do the most computation. The typical pattern: early/late layers are dim, middle layers are bright.
Difference Map — Where the first prompt gets more work (red) vs. the second prompt (blue). This isolates the effect of the changed word.
